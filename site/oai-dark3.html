Most tools make you choose between easy and powerful.

Some builders are fast but limiting. Custom development gives you control, but it is expensive and slow to change. Webdata Pro is built for the middle ground: launch quickly, then keep shaping the design, permissions, and data structure yourself.

Member groups
Not just a login wall.

Different kinds of members can have completely different experiences — different views, different permissions, different data. You define the groups; Webdata Pro enforces the rules.

A member can even be limited to only the records they created — so staff can see everything, but regular users only ever see their own.

Public visitor

No login required

Sees the public-facing search view. Maybe a teaser — enough to show the value, not enough to give it all away.

Full member

Logged in, verified

Sees the complete view — all fields, download links, contact info, whatever you decide is behind the gate.

Staff group

Elevated permissions

Can create, edit, and delete any record. Can see fields that are hidden from regular members. Gets a different homepage entirely.

Embed
A few lines of code. On your website.

Copy the embed block from the preview page and paste it into any HTML page. Your site's CSS handles the styling, so the experience feels like part of your brand instead of a bolted-on tool.

<script src="https://yourserver.com/static/embed.js"></script>
<div id="wdp-browse"></div>
<script>
  WDP.mount('#wdp-browse', {
    app: 'real-estate', view: 'browse-properties',
    baseUrl: 'https://yourserver.com'
  });
</script>
Built for the real world. Add your website's address to Allowed Origins once, paste the embed code, and your searchable view can live right inside your existing site.
The rest
The rest is pretty boring.
Which is exactly what you want from infrastructure.

🗄️ SQLite or MySQL

Start with SQLite — zero setup, single file, runs anywhere. Connect to MySQL when you need multi-user production. The admin works identically either way.

🔗 Visual join builder

Define many-to-one relationships between tables. Webdata Pro generates a live ERD diagram — you can even drag the tables around. As visual as the back end gets.

📬 Email notifications

Get alerted when records are created or changed. Immediate or daily digest. Configure it with your own SMTP credentials — your mail server, your inbox.

🔒 Security best practices

The standard security stuff, done correctly so you don't have to think about it: two-factor auth, password reset, session management, CSRF protection, and XSS sanitization.

📋 Public forms

Let anonymous visitors submit records — no login required. Optional reCAPTCHA keeps spam out. Works with $form[] tokens in any template.

🖥️ SQL Console

Run queries directly against your app database from the admin panel. For when you need to do something the UI doesn't have a button for.

Need a faster starting point? There’s a blueprint maker.

Describe what you’re building in plain English. It generates your tables, fields, groups, views, and sample data — a working starting point in under a minute. You don’t have to use it, but it’s a fast way to get past the blank-slate problem. Works with any LLM: local models via Ollama, or Claude, GPT-4, Gemini — your choice.

Get started
Five commands. No cloud account needed.

No Docker, no credit card, no deployment pipeline. Node.js and a terminal.

zsh — webdata-pro 
# Clone and install
$ git clone https://github.com/rytoplay/webdata-pro.git
$ cd webdata-pro && npm install

# Configure and start
$ cp .env.example .env
$ npm run dev

# Open http://localhost:3456/admin
